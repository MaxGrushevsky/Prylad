# Инструкция по деплою на Vercel

## Как Vercel определяет проекты?

Vercel определяет проекты по **Git репозиторию**. Есть два сценария:

### Сценарий 1: Отдельный репозиторий (рекомендуется)
- Если `shortlink` находится в **отдельном репозитории** (отличном от вашего первого проекта)
- Vercel автоматически создаст **новый отдельный проект**
- Каждый репозиторий = отдельный проект в Vercel

### Сценарий 2: Один репозиторий, несколько проектов
- Если оба проекта в **одном репозитории** (например, в монорепозитории)
- Можно создать **несколько проектов** из одного репозитория
- Каждый проект будет иметь свои настройки (root directory, build settings и т.д.)

## Создание второго проекта

### Через веб-интерфейс:
1. Нажмите **"Add New Project"** (даже если у вас уже есть проект)
2. Выберите репозиторий `shortlink`
3. Если репозиторий уже подключен, Vercel спросит:
   - **"Import"** - создать новый проект из этого репозитория
   - Или выберите другую ветку/директорию для нового проекта
4. Дайте проекту **уникальное имя** (например, `shortlink-tools`)

### Через CLI:
При первом деплое CLI спросит:
- **"Link to existing project?"** → Выберите **"No"** для создания нового проекта
- **"Project name?"** → Введите уникальное имя (например, `shortlink-tools`)

## Способ 1: Деплой через веб-интерфейс Vercel (Рекомендуется)

### Шаг 1: Подготовка репозитория
1. Убедитесь, что все изменения закоммичены и запушены в GitHub/GitLab/Bitbucket:
   ```bash
   git add .
   git commit -m "Prepare for deployment"
   git push origin main
   ```

### Шаг 2: Создание проекта на Vercel
1. Перейдите на [vercel.com](https://vercel.com)
2. Войдите через GitHub/GitLab/Bitbucket
3. Нажмите **"Add New Project"** (даже если у вас уже есть другие проекты)
4. Выберите ваш репозиторий `shortlink`
   - Если репозиторий уже используется другим проектом, Vercel предложит создать новый проект из него
   - Выберите **"Import"** или **"Create Project"**
5. Дайте проекту **уникальное имя** (например, `shortlink-tools` или `shortlink-generator`)
6. Vercel автоматически определит настройки:
   - **Framework Preset**: Next.js
   - **Build Command**: `npm run build` (или `next build`)
   - **Output Directory**: `.next` (автоматически)
   - **Install Command**: `npm install`

### Шаг 3: Настройка переменных окружения (если нужно)
Если у вас есть переменные окружения, добавьте их в настройках проекта:
- Settings → Environment Variables

### Шаг 4: Деплой
1. Нажмите "Deploy"
2. Дождитесь завершения сборки (обычно 2-5 минут)
3. После успешного деплоя вы получите URL вида: `https://your-project.vercel.app`

### Шаг 5: Настройка домена (опционально)
1. В настройках проекта перейдите в "Domains"
2. Добавьте ваш домен (например, `shortlink-qr.com`)
3. Следуйте инструкциям для настройки DNS

## Способ 2: Деплой через Vercel CLI

### Шаг 1: Установка Vercel CLI
```bash
npm install -g vercel
```

### Шаг 2: Вход в Vercel
```bash
vercel login
```

### Шаг 3: Деплой
```bash
# Первый деплой (preview)
vercel

# Продакшн деплой
vercel --prod
```

### Шаг 4: Настройка проекта
При первом деплое CLI задаст вопросы:
- **Set up and deploy?** → **Yes**
- **Which scope?** → Выберите ваш аккаунт/команду
- **Link to existing project?** → **No** (важно! Выберите "No" для создания нового проекта)
- **What's your project's name?** → Введите уникальное имя (например, `shortlink-tools`)
- **In which directory is your code located?** → **./** (текущая директория)
- **Want to override the settings?** → **No** (для стандартного Next.js проекта)

**Важно:** Если вы случайно выберете "Yes" на "Link to existing project?", вы можете:
- Отменить и запустить `vercel` снова
- Или вручную указать проект: `vercel link` → выбрать существующий проект или создать новый

## Важные замечания

### Обновление URL в robots.ts и sitemap.ts
После деплоя обновите URL в файлах:
- `app/robots.ts` - замените `https://shortlink-qr.com` на ваш реальный домен
- `app/sitemap.ts` - замените `https://shortlink-qr.com` на ваш реальный домен

Или используйте переменные окружения:
```typescript
const baseUrl = process.env.NEXT_PUBLIC_BASE_URL || 'https://your-domain.vercel.app'
```

### Проверка перед деплоем
1. Убедитесь, что проект собирается локально:
   ```bash
   npm run build
   ```
2. Проверьте, что нет ошибок линтера:
   ```bash
   npm run lint
   ```

### Автоматические деплои
После первого деплоя Vercel автоматически будет деплоить:
- **Preview deployments** - при каждом push в любую ветку
- **Production deployment** - при push в `main` (или настроенную production ветку)

## Troubleshooting

### Ошибки сборки
- Проверьте логи в Vercel Dashboard
- Убедитесь, что все зависимости указаны в `package.json`
- Проверьте версию Node.js (Vercel использует Node.js 18.x по умолчанию)

### Проблемы с путями
- Убедитесь, что `tsconfig.json` правильно настроен
- Проверьте импорты в коде

### Переменные окружения
- Не забывайте добавлять переменные окружения в настройках проекта Vercel
- Используйте `NEXT_PUBLIC_` префикс для переменных, доступных в браузере

## Управление проектами

### Проверка текущего проекта
```bash
# Посмотреть, к какому проекту привязана текущая директория
vercel ls

# Посмотреть все ваши проекты
vercel projects ls
```

### Привязка к существующему проекту
Если вы хотите привязать локальную директорию к существующему проекту:
```bash
vercel link
```
CLI спросит:
- **Which scope?** → Выберите аккаунт
- **Link to existing project?** → **Yes**
- **What's the name of your existing project?** → Выберите проект из списка

### Создание нового проекта через link
```bash
vercel link
```
- **Link to existing project?** → **No**
- Введите имя нового проекта

### Отвязка от проекта
```bash
# Удалить связь с проектом (удалит .vercel директорию)
rm -rf .vercel
```

## Полезные команды

```bash
# Просмотр деплоев текущего проекта
vercel ls

# Просмотр всех ваших проектов
vercel projects ls

# Просмотр логов
vercel logs

# Удаление проекта
vercel remove

# Информация о текущем проекте
vercel inspect
```
